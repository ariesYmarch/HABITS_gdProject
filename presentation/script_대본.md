## 3. 목표 및 해결방안 (Engineer View)

### 솔루션 1: 사용자 맞춤 습관 추천

> "첫 번째 솔루션은 사용자 맞춤 습관 추천 시스템입니다.
>
> Feature 1-1, 성격 유형 기반 습관 매칭입니다.
> 저희는 4개 카테고리에 걸쳐 총 80개의 성격 해시태그를 설계했고, 이 조합으로 22개의 성격 유형을 정의했습니다. 사용자는 온보딩 과정에서 '현재의 나'와 '이상적인 나' 두 가지 성격 테스트를 수행하게 되는데요, 이 둘의 Gap을 분석하여 부족한 성격 특성을 강화할 수 있는 습관을 우선적으로 추천합니다.
>
> Feature 1-2는 상황 인식 기반 추천입니다.
> 사용자의 라이프스타일, 예를 들어 통근 시간이나 취침 전 같은 22개 상황 컨텍스트를 분석해서, 그 상황에서 물리적으로 수행 가능한 습관만 추천합니다. 통근 중에는 오디오 습관에 2배 가중치를 주고, 신체 활동에는 페널티를 주는 식입니다. 이 추천 엔진은 Recombee API와 연동할 수 있도록 설계했고, 신규 사용자의 콜드 스타트 문제는 해시태그 기반 콘텐츠 필터링으로 해결합니다.
>
> Feature 1-3으로는 10개 카테고리에 걸친 124개 습관 템플릿 데이터베이스를 구축했습니다. 각 템플릿에는 난이도, 수행 상황, 강화 태그, 예상 소요시간이 매핑되어 있어서 다차원 스코어링이 가능합니다."

---

### 솔루션 2: 감정-습관 통합 관리

> "두 번째 솔루션은 감정과 습관을 통합 관리하는 시스템입니다.
>
> Feature 2-1, 일기 기반 감정 분석 엔진입니다.
> 사용자가 작성한 한국어 일기 텍스트에서 8종의 감정을 자동으로 분류합니다. 분석은 3단계로 이루어지는데요, 먼저 감정별 키워드 사전으로 텍스트를 스캔하고, 한국어 특유의 표현인 'ㅠ'나 느낌표, 물음표 같은 부호로 감정 점수를 보정한 다음, 최종 정규화를 거쳐 주요 감정을 도출합니다. 백엔드에서는 HuggingFace의 KoELECTRA 모델을 연동할 수 있도록 설계했습니다.
>
> Feature 2-2는 감정과 이행률의 상관관계 분석입니다.
> 일기 작성 시점의 감정과 해당 주간의 습관 이행률을 병렬로 추적합니다. 부정 감정이 빈번한 주간에는 이행률이 하락하는 패턴이 감지되면, 이를 적응형 목표 조정의 트리거로 활용합니다.
>
> Feature 2-3은 이 모든 것을 연결하는 4단계 습관 코칭 파이프라인입니다. 온보딩에서 시작해서 상황 인식 추천, 이행 관리, 그리고 동적 피드백까지 하나의 순환 구조로 작동합니다."

---

### 솔루션 3: 데이터 기반 피드백

> "세 번째 솔루션은 데이터 기반의 AI 피드백 시스템입니다.
>
> Feature 3-1, AI 적응형 코칭 메시지 생성입니다.
> OpenAI GPT-4o-mini를 활용하여, 사용자의 이행률과 감정 상태를 종합한 맞춤 코칭 메시지를 자동 생성합니다. 프롬프트는 자기결정이론, SDT에 기반해서 설계했는데요, 핵심은 사용자의 '실패'를 '조정'으로 리프레이밍하는 공감적 톤입니다. 또한 API 장애 시에도 앱이 정상 작동하도록 로컬 규칙 기반 폴백 구조를 갖추고 있습니다.
>
> Feature 3-2는 70% 임계값 기반의 동적 목표 조정입니다.
> 행동 변화 심리학에서 70에서 80%가 '충분한 반복'의 기준인데요, 이를 기반으로 이행률이 30% 미만이면 목표를 하향하고, 70% 이상이면 난이도를 상향 제안합니다. 특히 부정 감정이 감지되면 임계값을 완화하여, 심리적 부담을 최소화하면서 습관을 유지할 수 있도록 합니다.
>
> Feature 3-3은 성장 데이터 시각화입니다.
> 주간, 월간 이행률과 감정 변화 추이를 대시보드로 제공하고, 캘린더 뷰에서 습관 완료 여부와 감정 이모지를 동시에 보여줘서 사용자의 자기 성찰을 유도합니다."

## 4. 시스템 아키텍처

> "시스템 아키텍처입니다.
>
> 프론트엔드는 React Native와 TypeScript로 개발했고, Zustand으로 상태관리, AsyncStorage로 로컬 데이터를 영속 저장합니다. 화면 전환은 React Navigation v7의 네이티브 스택을 사용합니다.
>
> 백엔드는 FastAPI 기반으로, 감정 분석과 피드백 생성, 추천 로직을 처리합니다. SDT 기반의 70% 임계값 피드백 알고리즘이 여기서 동작합니다.
>
> 데이터베이스는 PostgreSQL를 사용하여 사용자, 성격 유형, 습관, 일기, 감정 데이터를 저장합니다.
>
> AI 모델은 OpenAI GPT-4o-mini로 적응형 코칭 메시지를 생성하고, HuggingFace의 KoELECTRA로 한국어 감정 분류를 수행합니다. API 장애 시에는 로컬 규칙 기반으로 자동 전환되는 폴백 구조를 갖추고 있습니다."

## 5. 개발 현황

> "개발 현황입니다.
>
> 설계 단계에서는 React Native, FastAPI, PostgreSQL 기반의 3-Tier 아키텍처 설계를 완료했고, 80개 해시태그와 22개 성격 유형, 124개 습관 템플릿의 데이터 모델링을 완료했습니다.
>
> 개발 단계에서는 프론트엔드의 핵심 화면 구현을 완료했고, 백엔드의 감정 분석, 피드백 생성, 추천 API를 구현했습니다. OpenAI와 HuggingFace API의 시험 연동도 완료한 상태입니다.
>
> 검증 단계에서는 iOS 시뮬레이터 빌드 테스트를 통과했고, 현재 실 사용자 대상 Usability Test와 배포 환경 구축을 준비하고 있습니다."
